## Required attribution

Use this credit, unmodified, wherever your app shows data sources or credits
(an About/Settings screen, a footer, an onboarding "powered by" line, etc.):

```

```

Link it to **https://github.com/jtvargas/merchant-studio** wherever your UI supports
links. If the surface has no room for a link (e.g. a native app's plain-text credits
list), the text alone satisfies the requirement.

### Short form (tight UI space)

If you're truly constrained on space (e.g. a single-line footer), this shorter form is acceptable:

```

```

## Why this satisfies CC BY 4.0

Section 3(a) of the license requires you to retain, "to the extent reasonably
practicable," identification of the creator and a way back to the license. The
strings above identify the creator (Jonathan Taveras) and the source (Merchant
Studio); linking to the repository satisfies the reference to the license and
material. You do not need to reproduce the full license text inside your app —
a link to the repository (which contains `LICENSE`) is enough.

## What you don't need to do

- No requirement to open-source your app or any code that uses this data.
- No share-alike obligation — you may keep your own derived/modified data private.
- No royalties, no permission request, no notifying the author.
